Control Structures in C

Mastering decision-making and loops to solve real-world problems in C.

Govardhan Gurale

Team Member

In standard C programming, statements are executed sequentially—one after the other. However, real-world problems require the program to make choices or repeat tasks. Control structures allow us to break this sequential flow.

1. Decision Making (Branching)

Decision-making structures require the programmer to specify one or more conditions to be evaluated. In the MSBTE syllabus, we focus on if-else and the switch statement.

[Image of if-else flowchart in C programming]
int marks = 75; if (marks >= 40) { printf("Result: Pass\n"); } else { printf("Result: Fail\n"); }

2. The Switch Statement

When you have multiple choices (like a menu-driven program), a long chain of if-else statements becomes messy. The switch statement is a cleaner alternative for multi-way branching.

switch(choice) { case 1: printf("Addition"); break; case 2: printf("Subtraction"); break; default: printf("Invalid Choice"); }

3. Loop Control (Iteration)

Loops are used to execute a block of code multiple times. This is vital for tasks like printing a table, calculating factorials, or processing array elements.

  • while: Entry-controlled loop. The condition is checked before execution.
  • do-while: Exit-controlled loop. The code runs at least once before checking the condition.
  • for: Most commonly used when the number of iterations is known in advance.
// Printing numbers from 1 to 5 for(int i = 1; i <= 5; i++) { printf("%d ", i); }
"Tip: Always use a 'break' statement in each switch case unless you want the code to 'fall through' to the next case!"

By mastering these control structures, you move from writing simple scripts to building logic-driven applications. In our next article, we will look at how loops and decision-making come together to handle Arrays.
